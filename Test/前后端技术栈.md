技术栈最终总结（无代码版）
后端（Python）
框架：FastAPI
核心作用：基于 Python 类型提示自动生成交互式 API 文档（/docs 和 /openapi.json），接口定义变更时无需手动维护文档
数据模型：Pydantic
核心作用：强制类型校验，确保请求/响应数据与蓝图设计一致，错误数据在接口层直接拦截
API 文档：FastAPI 原生 OpenAPI 生成
核心作用：作为前后端唯一的机器可读契约，Orval 直接读取此文件生成前端代码
前端（React TypeScript）
HTTP 客户端：Axios
核心作用：基础请求库，由 Orval 自动封装，业务代码不直接接触 Axios
API 客户端生成：Orval
核心作用：读取后端的 /openapi.json，自动生成：
纯函数 API 调用（workflowApi.ts）
React Query Hooks（workflow.hooks.ts）
TypeScript 类型定义（workflow.types.ts）
跨会话机制：src/api/generated/ 目录不提交 Git，每次新会话运行 npm run generate:api 动态生成，保证与后端实时同步
状态管理 + 异步轮询：TanStack Query (v5)
核心作用：自动处理 generating/uploading/upload_complete 状态的轮询，缓存管理，乐观更新
UI 组件库：Ant Design v5
核心作用：表格、表单、标签、上传、Modal 等开箱即用，无需自定义复杂组件
运行时校验（自动生成）：Zod
核心作用：Orval 自动集成，对后端返回数据做运行时校验，防止字段缺失导致白屏，错误信息直接定位到字段级别
前后端协作核心机制
真理源：FastAPI 的 /openapi.json 是唯一机器可读契约
同步命令：前端 npm run generate:api 一键同步所有 API 和类型
类型安全：TypeScript 编译时自动检测字段变更，所有引用处必须适配
跨会话自洽：每次新会话只需 git pull + npm run generate:api，无需回忆任何上下文
